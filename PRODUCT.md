# Product

## Register

product

## Users

Researchers and analysts studying AI interaction patterns at scale. They work with session traces to understand model behavior, debug interactions, or analyze usage patterns across many conversations. Context is analytical work requiring clear data hierarchy and efficient drill-down workflows.

## Product Purpose

A web viewer for OpenCode trace data that enables systematic exploration of AI sessions. Success means researchers can quickly navigate from session overviews to specific conversations, understand token usage patterns, and export data for further analysis — without being overwhelmed by raw data volume.

## Brand Personality

Technical-neutral: clean, precise, no distraction. Like a debugger or profiler UI — function-first, no marketing decoration. Trustworthy accuracy over visual appeal.

## Anti-references

Raw data dumps — interfaces that present overwhelming tables without hierarchy, no clear focus, requiring users to mentally parse structure. Avoid generic data tables that dump everything equally.

## Design Principles

1. **Show hierarchy, not volume** — Structure information so users can navigate from overview to detail. Never dump raw data without clear pathways.
2. **Technical precision without decoration** — Every element serves a function. No gradient accents, hero metrics, or marketing-style presentation.
3. **Drill-down workflow** — Enable exploration from macro (session list) to micro (conversation blocks). Clear pathways, not dead ends.
4. **Scale-ready** — Designed for analyzing many sessions, not just one. Handle 100+ sessions without cognitive overload.
5. **Accessible by default** — WCAG AA compliance. Screen readers, keyboard navigation, color contrast all work from day one.

## Accessibility & Inclusion

WCAG 2.1 AA level compliance. Standard accessibility for public analytical tools: sufficient color contrast, keyboard navigation, screen reader support, semantic markup.
